# Issue Log: Login TypeError

- Date: 2026-07-05
- Severity: High
- Status: Open
- Owner: Backend
- Module: Users / Auth

## Summary
POST /api/auth/login/ returns 500 with TypeError: 'User' object is not subscriptable when attempting to obtain JWT tokens via SimpleJWT TokenObtainPairSerializer.validate().

## Steps to Reproduce
1. Start backend: python manage.py runserver 8001
2. Create superuser or existing user with email/password
3. POST /api/auth/login/ with {"email":"...","password":"..."}
4. Observe 500 TypeError instead of tokens JSON.

## Evidence
Traceback points to rest_framework_simplejwt/serializers.py line 46 inside validate, called from apps.users.views.login_view. The custom user is not being handled as a dict-like object during token serialization/validation.

## Impact
No user can log in via API. Frontend auth is blocked.
